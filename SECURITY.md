# Security Policy

## Reporting a Vulnerability

Please do not report security issues in public GitHub issues.

Use GitHub's private vulnerability reporting if it is enabled for the repository. If that is not available, contact the maintainers through a private channel and include:

- a short description of the issue
- affected files or features
- steps to reproduce
- possible impact
- any suggested mitigation

Reports will be reviewed as quickly as possible, and confirmed issues should be fixed before public disclosure.
